# `D_latch.vhdl` 줄별 설명

## 먼저: D-latch가 뭐냐?

D-latch는 `Data Latch`라고 부르는 기본 저장 소자입니다.  
디지털 회로에서 1비트 값을 잠시 붙잡아 두는 가장 단순한 형태 중 하나입니다.

아주 짧게 말하면:

- `d`: 저장하려는 입력값
- `en`: 지금 입력을 통과시킬지 정하는 신호
- `q`: 현재 출력값

핵심은 `en`이 켜져 있을 때만 입력 `d`가 출력 `q`로 전달된다는 점입니다.

## 래치는 왜 필요한가?

조합 회로는 입력이 바뀌면 출력도 바로 바뀝니다.

그런데 어떤 회로는 값을 그냥 바로 흘려보내지 말고,  
특정 조건에서만 받아들이는 동작이 필요합니다.

예를 들어:

- 어떤 제어 신호가 켜졌을 때만 데이터를 통과시키고 싶을 때
- 다음 단계 회로에 값을 잠시 유지해서 넘기고 싶을 때
- 플립플롭의 내부 구조를 이해하고 싶을 때

이런 경우 래치 개념이 기본이 됩니다.

## D-latch를 직관적으로 보면

D-latch는 문이 달린 통로와 비슷합니다.

- `en='1'`이면 문이 열려 있어서 입력 `d`가 그대로 지나감
- `en='0'`이면 문이 닫혀 있어서 이전 값이 유지됨

그래서 D-latch는 "열려 있을 때는 따라가고, 닫히면 마지막 값을 유지하는 회로"라고 이해하면 쉽습니다.

## 이 파일의 역할

이 파일은 가장 기본적인 레벨 민감형 `D-latch`를 VHDL로 표현한 예제입니다.  
`enable` 신호가 1이면 입력을 통과시키고, 0이면 기존 출력을 유지합니다.

## 원문 코드

```vhdl
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;

entity D_latch is
    Port (
        en : in  STD_LOGIC;
        d  : in  STD_LOGIC;
        q  : out STD_LOGIC
    );
end D_latch;

architecture Behavioral of D_latch is
begin

    process(en, d)
    begin
        if en = '1' then
            q <= d;
        end if;
    end process;

end Behavioral;
```

## 큰 구조 먼저 보기

이 파일은 아래 네 부분으로 볼 수 있습니다.

1. 라이브러리 선언
2. `entity`에서 입출력 선언
3. `process`로 래치 동작 정의
4. `architecture` 종료

## 이 코드가 하는 핵심 동작

이 회로의 핵심은 아주 단순합니다.

- `en='1'`이면 `q <= d`
- `en='0'`이면 아무 값도 새로 쓰지 않음

즉, `en`이 꺼져 있으면 출력 `q`는 마지막으로 저장된 값을 유지합니다.

이런 특성 때문에 래치는 조합 회로가 아니라 저장 성질을 가진 순차 회로 요소로 볼 수 있습니다.

## 줄별 해설

### 1-2. 라이브러리

- `library IEEE;`
  표준 IEEE 라이브러리를 사용하겠다는 선언입니다.
- `use IEEE.STD_LOGIC_1164.ALL;`
  `STD_LOGIC` 타입과 논리 신호 관련 정의를 가져옵니다.

### 4. `entity D_latch is`

회로의 바깥 모양을 선언합니다.  
이 회로 이름은 `D_latch`입니다.

### 5-9. `Port (...)`

입출력 포트를 정의합니다.

- `en`
  래치를 열고 닫는 enable 입력입니다.
- `d`
  데이터 입력입니다.
- `q`
  저장되거나 전달된 결과 출력입니다.

### 10. `end D_latch;`

`entity` 선언이 끝났다는 뜻입니다.

### 12. `architecture Behavioral of D_latch is`

회로 내부 동작을 적는 부분입니다.

- `Behavioral`은 구조 이름
- `of D_latch`는 이 구조가 `D_latch` 엔티티용이라는 뜻입니다.

### 13. `begin`

실제 동작 설명이 시작됩니다.

### 15-19. 래치 동작

```vhdl
process(en, d)
begin
    if en = '1' then
        q <= d;
    end if;
end process;
```

이 부분이 D-latch의 핵심입니다.

- `process(en, d)`
  `en` 또는 `d`가 바뀌면 다시 평가합니다.
- `if en = '1' then`
  enable이 1일 때만 입력을 받아들입니다.
- `q <= d;`
  입력 `d`를 출력 `q`로 전달합니다.

반대로 `en='0'`일 때는 `q`에 새 값을 할당하지 않기 때문에,  
출력은 이전 값을 계속 유지하게 됩니다.

이것이 바로 래치 동작입니다.

### 21. `end Behavioral;`

`architecture`가 끝났다는 뜻입니다.

## 래치와 플립플롭 차이

처음 배울 때 가장 중요한 구분입니다.

### 래치

래치는 특정 레벨 동안 열려 있습니다.

예를 들어 이 코드에서는:

- `en='1'`인 동안 입력이 계속 출력에 반영됩니다.

즉, enable이 열린 시간 동안은 입력 변화가 출력으로 그대로 전달될 수 있습니다.

### 플립플롭

플립플롭은 보통 클럭의 상승 에지나 하강 에지 같은  
"순간"에만 값을 저장합니다.

그래서 동기식 시스템에서는 보통 플립플롭을 더 많이 씁니다.

### 연결해서 보면

- D-latch는 "레벨에 민감"
- DFF는 "에지에 민감"

이라고 구분하면 이해하기 쉽습니다.

## 이 코드에서 중요한 포인트

이 코드에서 `else`가 없는 것은 실수가 아니라 래치를 의도한 표현입니다.

만약 `en='0'`일 때도 다른 값을 대입해 버리면,  
출력을 유지하는 저장 특성이 사라질 수 있습니다.

즉:

- 값을 일부 조건에서만 대입
- 나머지 경우에는 유지

이 패턴이 래치를 만들게 됩니다.

## 따라칠 때 기억할 문법

- `process(...)`는 입력 변화에 반응하는 동작 블록
- `if en = '1' then`은 enable 조건
- `<=`는 신호 할당
- 조건에 따라 일부 경우만 값을 쓰면 저장 동작이 생길 수 있음

## 주의해서 볼 점

래치는 입력이 열려 있는 동안 계속 출력이 흔들릴 수 있기 때문에,  
실무 동기식 설계에서는 플립플롭 기반 설계를 더 선호하는 경우가 많습니다.

하지만 래치는:

- 저장 소자의 기본 개념을 이해하는 데 좋고
- 마스터-슬레이브 플립플롭 구조를 이해하는 출발점이 되며
- 일부 저전력/특수 설계에서는 실제로 쓰이기도 합니다.

## 한 문장으로 다시 정리

D-latch는 `enable`이 켜져 있을 때 입력을 통과시키고, 꺼지면 마지막 값을 유지하는 가장 기본적인 1비트 저장 소자입니다.
