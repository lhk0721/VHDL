# `DFF.vhdl` 줄별 설명

## 먼저: DFF가 뭐냐?

DFF는 `D Flip-Flop`의 줄임말입니다.  
디지털 회로에서 "값을 1비트 저장하는 가장 기본적인 기억 소자"라고 보면 됩니다.

입력이 들어오면 그 값을 그냥 바로 내보내는 것이 아니라,  
특정한 클럭 타이밍에 값을 받아서 기억해 두었다가 출력합니다.

아주 짧게 말하면:

- `d`: 지금 들어오는 값
- `clk`: 언제 저장할지 알려 주는 기준 신호
- `q`: 저장되어 있는 값

즉, DFF는 "값을 잠깐 기억하는 1비트 메모리"입니다.

## 왜 DFF가 필요한가?

조합 회로만 있으면 입력이 바뀌는 즉시 출력도 바뀝니다.

예를 들어 반가산기 같은 회로는:

- 입력이 바뀌면
- 계산 결과도 바로 바뀝니다.

이런 회로는 계산은 할 수 있지만, "이전 상태를 기억"하지는 못합니다.

그런데 실제 디지털 시스템은 기억이 꼭 필요합니다.

예를 들어:

- 카운터는 이전 숫자를 기억해야 다음 숫자를 만들 수 있습니다.
- 상태기계는 현재 상태를 기억해야 다음 상태로 넘어갈 수 있습니다.
- 레지스터는 데이터를 잠시 저장해야 합니다.
- CPU도 중간 계산 결과를 저장해야 합니다.

이때 가장 기본이 되는 저장 소자가 DFF입니다.

즉:

- 계산만 하는 회로의 기본은 조합 회로
- 기억까지 하는 회로의 기본은 플립플롭

라고 생각하면 됩니다.

## DFF를 일상적으로 비유하면

DFF는 종이에 적힌 값을 책상 위에 그냥 펼쳐 두는 것이 아니라,  
종이함에 넣어 두고 정해진 시간에만 새 종이로 교체하는 것과 비슷합니다.

- `d`는 새로 들어온 종이
- `clk`는 "지금 교체하세요"라는 신호
- `q`는 현재 보관 중인 종이

클럭 신호가 없으면 회로가 제멋대로 흔들릴 수 있는데,  
DFF를 쓰면 "이 타이밍에만 값을 확정하자"라고 기준을 세울 수 있습니다.

## 이 파일의 역할

이 파일은 D 플립플롭(D Flip-Flop)을 VHDL로 표현한 예제입니다.  
입력 `d` 값을 클럭에 맞춰 받아서 출력 `q`에 저장합니다.

이 코드는 내부적으로 래치 두 개를 이어 붙인 마스터-슬레이브 구조로 작성되어 있습니다.

## 원문 코드

```vhdl
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;

entity DFF is
    Port (
        clk : in  STD_LOGIC;
        d   : in  STD_LOGIC;
        q   : out STD_LOGIC
    );
end DFF;

architecture Behavioral of DFF is
    signal master_q : STD_LOGIC;
begin

    -- Master D latch (enable = not clk)
    process(clk, d)
    begin
        if clk = '0' then
            master_q <= d;
        end if;
    end process;

    -- Slave D latch (enable = clk)
    process(clk, master_q)
    begin
        if clk = '1' then
            q <= master_q;
        end if;
    end process;

end Behavioral;
```

## 큰 구조 먼저 보기

이 파일은 아래 네 부분으로 볼 수 있습니다.

1. 라이브러리 선언
2. `entity`에서 입출력 선언
3. 내부 저장 신호 `master_q` 선언
4. 마스터 래치와 슬레이브 래치 프로세스

## 이 코드가 표현하는 핵심 동작

DFF의 핵심은 "입력을 아무 때나 반영하지 않고, 클럭에 맞춰 저장한다"는 점입니다.

예를 들어 `d` 값이 계속 바뀌더라도:

- 어떤 순간에는 아직 저장하지 않고 지켜보다가
- 클럭 조건이 맞으면
- 그때 값을 받아서
- 출력 `q`를 유지합니다

그래서 출력이 안정적이 됩니다.

이게 순차 회로의 출발점입니다.

## 래치와 플립플롭은 어떻게 다른가?

처음 배우면 래치와 플립플롭이 헷갈리기 쉽습니다.

### 래치

래치는 "열려 있는 동안" 입력이 출력에 반영됩니다.

예를 들어 enable이 1인 동안 계속 입력을 따라가면,  
그건 입력이 잠깐만 흔들려도 출력도 같이 흔들릴 수 있습니다.

### 플립플롭

플립플롭은 보통 클럭의 특정 순간에만 값을 받아들입니다.

그래서 래치보다 더 안정적으로 설계하기 쉽고,  
동기식 회로에서 표준처럼 많이 사용됩니다.

### 이 파일의 방식

현재 `DFF.vhdl`은 래치 두 개를 조합해서 플립플롭처럼 동작하게 만든 예제입니다.

- 마스터 래치가 한쪽 클럭 구간에서 입력을 받음
- 슬레이브 래치가 반대쪽 클럭 구간에서 그 값을 출력으로 넘김

그래서 결과적으로는 입력이 바로 출력으로 새어 나가지 않고,  
클럭 경계에 맞춰 값이 전달되는 효과가 납니다.

## 줄별 해설

### 1-2. 라이브러리

- `library IEEE;`
  표준 IEEE 라이브러리를 사용하겠다는 선언입니다.
- `use IEEE.STD_LOGIC_1164.ALL;`
  `STD_LOGIC` 타입과 기본 논리 신호 정의를 가져옵니다.

### 4. `entity DFF is`

회로의 바깥 모양을 선언합니다.  
이 회로의 이름은 `DFF`입니다.

### 5-9. `Port (...)`

입출력 포트를 정의합니다.

- `clk`
  클럭 입력입니다.
- `d`
  저장할 데이터 입력입니다.
- `q`
  저장된 결과 출력입니다.

`in`은 입력, `out`은 출력을 뜻합니다.

### 10. `end DFF;`

`entity` 선언이 끝났다는 뜻입니다.

### 12. `architecture Behavioral of DFF is`

회로 내부 동작을 작성하는 부분입니다.

- `Behavioral`은 이 구조의 이름
- `of DFF`는 이 구조가 `DFF` 엔티티에 연결된다는 뜻

### 13. 내부 신호 선언

```vhdl
signal master_q : STD_LOGIC;
```

마스터 래치의 출력을 저장하는 내부 신호입니다.  
즉, 입력 `d`를 바로 `q`로 보내지 않고 중간 단계 `master_q`를 거치게 합니다.

### 14. `begin`

실제 동작 설명이 시작됩니다.

### 16-21. 마스터 래치

```vhdl
process(clk, d)
begin
    if clk = '0' then
        master_q <= d;
    end if;
end process;
```

이 부분은 클럭이 `0`일 때 열리는 D 래치입니다.

- `process(clk, d)` 는 `clk` 또는 `d`가 바뀌면 다시 평가
- `if clk = '0' then`
  클럭이 낮은 동안만 입력을 통과
- `master_q <= d;`
  입력 `d`를 내부 신호 `master_q`에 저장

즉, 클럭이 `0`인 동안에는 `master_q`가 `d`를 따라갑니다.

### 23-28. 슬레이브 래치

```vhdl
process(clk, master_q)
begin
    if clk = '1' then
        q <= master_q;
    end if;
end process;
```

이 부분은 클럭이 `1`일 때 열리는 슬레이브 래치입니다.

- `clk='1'` 인 동안만 동작
- `master_q` 값을 최종 출력 `q`에 전달

그래서 전체적으로 보면:

- 클럭이 `0`일 때는 마스터가 입력을 받음
- 클럭이 `1`일 때는 슬레이브가 마스터 값을 출력으로 보냄

이 구조 때문에 겉보기에는 플립플롭처럼 동작합니다.

### 30. `end Behavioral;`

`architecture`가 끝났다는 뜻입니다.

## 이해 포인트

이 예제는 D 플립플롭을 이해하기 쉽게 래치 두 개로 나눠 표현한 코드입니다.

- 마스터 래치: `clk='0'`일 때 입력을 받음
- 슬레이브 래치: `clk='1'`일 때 내부 값을 출력으로 전달

그래서 입력이 계속 흔들려도 출력은 클럭 변화에 맞춰 한 단계 늦게 반영되는 효과를 냅니다.

## 왜 이런 구조가 중요한가?

디지털 회로를 크게 보면 대부분 아래 두 가지가 섞여 있습니다.

1. 현재 입력으로 계산하는 부분
2. 이전 값을 기억하는 부분

DFF는 그중 2번을 담당합니다.

예를 들어 카운터를 생각해 보면:

- 현재 값이 3인지 기억하고 있어야
- 다음 클럭에서 4로 바꿀 수 있습니다

만약 기억 장치가 없으면 회로는 "지금 이전에 몇이었는지"를 알 수 없습니다.

즉, DFF는 단순한 1비트 저장소이지만,  
여러 개가 모이면 레지스터가 되고,  
레지스터들이 모이면 카운터, 상태 저장부, 파이프라인 레지스터 같은 더 큰 구조가 됩니다.

그래서 DFF는 순차 회로의 벽돌 같은 존재입니다.

## 실제로 어디에 쓰이나?

DFF 하나만 단독으로 쓰기보다 여러 개를 묶어서 자주 사용합니다.

- `8개`를 묶으면 8비트 레지스터
- 여러 개를 이어서 쓰면 시프트 레지스터
- 피드백과 조합 회로를 붙이면 카운터
- 상태 비트로 쓰면 FSM의 상태 저장부

즉, 지금 보고 있는 `DFF.vhdl` 하나를 이해하면  
앞으로 배우는 카운터, 레지스터, 상태기계 이해가 훨씬 쉬워집니다.

## 입력 `d`, 출력 `q`, 클럭 `clk`를 다시 직관적으로 보기

- `d`: "다음에 저장하고 싶은 값"
- `clk`: "지금 저장할 타이밍"
- `q`: "현재 저장되어 있는 값"

핵심은 `q`가 항상 `d`와 같지는 않다는 점입니다.

`d`는 계속 변할 수 있지만,  
`q`는 저장 타이밍이 오기 전까지 이전 값을 유지할 수 있습니다.

이 차이 때문에 회로가 질서 있게 움직입니다.

## 따라칠 때 기억할 문법

- `process(...)` 안은 순차적으로 읽는 구조
- `if clk = '0' then`, `if clk = '1' then` 처럼 조건으로 래치 동작을 표현 가능
- `signal`은 회로 내부 연결용 신호
- `<=` 는 신호 할당

## 주의해서 볼 점

실무에서는 보통 D 플립플롭을 아래처럼 상승 에지 기준으로 더 직접 표현하는 경우가 많습니다.

```vhdl
process(clk)
begin
    if rising_edge(clk) then
        q <= d;
    end if;
end process;
```

현재 `DFF.vhdl`은 학습용으로 마스터-슬레이브 구조를 드러내는 예제라고 보면 이해하기 쉽습니다.

## 한 문장으로 다시 정리

DFF는 "클럭 타이밍에 맞춰 1비트 값을 저장하는 기억 소자"이고,  
디지털 시스템이 이전 값을 기억하면서 순서대로 동작하게 만드는 가장 기본적인 부품입니다.
